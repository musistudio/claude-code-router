## 工具定位

**生图工具** 是 Fusion 的内置媒体工具。它负责把模型调用转成图片生成或图片编辑请求，再把产物路径、MIME、大小、SHA-256 和限时访问 URL 返回给基础模型。

它和 **内置图像能力** 不同：

| 能力 | 作用 |
| --- | --- |
| 内置图像能力 | 理解用户提供的图片、截图、图表和 OCR 内容。 |
| 生图工具 | 生成新图片，或基于 1 到 3 张本地图片进行编辑。 |

## 配置步骤

1. 在 **供应商** 页面配置支持图片生成协议的供应商和模型。
2. 新建或编辑 Fusion 模型。
3. 在 **Tools** 中添加 **图片生成**。
4. 在工具配置里选择图片模型，例如 `供应商/模型`。
5. 保存 Fusion 模型，并把它选为 Agent 模型或路由目标。

导入 Grok Agent 后，CCR 会自动提供 `grok-imagine-image-quality`。ai-gateway 会复用已有 OAuth 登录态访问 `api.x.ai`，不会额外启动 Grok CLI。

## 支持的请求

CCR 通过 ai-gateway 的通用媒体协议调用供应商：

| 请求 | 说明 |
| --- | --- |
| `images/generations` | 文本生图。 |
| `images/edits` | 基于本地图片输入进行编辑。 |

工具调用支持可选 `idempotency_key`。同一次用户意图应复用稳定的 Key，避免网络重试导致重复计费。

## 本地图片输入

图片编辑会校验本地输入的真实路径、文件头和大小。CCR 默认允许范围明确的当前工作目录、系统临时目录和 CCR 配置目录。

不要把文件系统根目录、用户主目录或其上级目录隐式放开。确实需要扩大读取范围时，再显式配置 `allowedInputRoots`。

## 产物

生成图片保存在 CCR 私有数据目录。返回结果包含：

- 本地文件路径
- MIME 类型
- 文件大小
- SHA-256
- 限时访问 URL

限时 URL 用独立 token 保护，不直接复用 CCR API Key。

## 排查要点

生图失败时，优先检查：

- 供应商模型是否声明或实际支持图片生成。
- Fusion 工具里是否为图片生成选择了正确模型。
- 请求日志中 ai-gateway 返回的状态码和错误信息。
- 图片编辑输入文件是否位于允许读取的目录内。
