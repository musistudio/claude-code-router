## 工具定位

**生视频工具** 是 Fusion 的内置媒体工具。它适合把文本模型升级成能提交视频生成任务的 Agent 模型，例如根据需求描述、图片输入或参考图创建视频。

视频生成始终按异步任务执行。启动调用会立即返回 Job ID，随后由工具查询任务状态，直到完成、失败或被取消。

## 配置步骤

1. 在 **供应商** 页面配置支持视频生成协议的供应商和模型。
2. 新建或编辑 Fusion 模型。
3. 在 **Tools** 中添加 **视频生成**。
4. 在工具配置里选择视频模型，例如 `供应商/模型`。
5. 保存 Fusion 模型，并把它选为 Agent 模型或路由目标。

导入 Grok Agent 后，CCR 会自动提供 `grok-imagine-video`。ai-gateway 会复用已有 OAuth 登录态访问 `api.x.ai`，不会额外启动 Grok CLI。

## 支持的请求

CCR 通过 ai-gateway 的通用媒体协议调用供应商：

| 请求 | 说明 |
| --- | --- |
| `videos/generations` | 启动文本生视频、图生视频或参考图生视频任务。 |
| `videos/{id}` | 查询视频任务状态和结果。 |

工具调用支持可选 `idempotency_key`。同一次用户意图应复用稳定的 Key，避免网络重试导致重复计费。

## 运行时行为

生视频工具会把任务生命周期交给 CCR 管理：

- 启动任务后返回 Job ID。
- 轮询任务状态并把进度反馈给模型。
- 任务完成后返回视频产物信息。
- 用户取消请求时，工具会尽量取消或停止继续等待。

请求超时和客户端取消仍然生效。并发、保留期和任务超时属于 CCR 内部安全策略，通常不需要在 Fusion UI 中配置。

## 产物

生成视频保存在 CCR 私有数据目录。返回结果包含：

- 本地文件路径
- MIME 类型
- 文件大小
- SHA-256
- 限时访问 URL

视频 URL 支持 HTTP Range，便于播放器按需加载。

## 排查要点

生视频失败时，优先检查：

- 供应商模型是否声明或实际支持视频生成。
- Fusion 工具里是否为视频生成选择了正确模型。
- 请求日志中 ai-gateway 返回的状态码和错误信息。
- 图生视频或参考图生视频的输入文件是否位于允许读取的目录内。
